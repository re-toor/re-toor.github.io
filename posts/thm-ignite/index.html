Tryhackme - Ignite

September 30, 2021 2-minute read
CTF • Tryhackme
web • linux • exploit-db • fuel-cms

Hello, this is Neo. Today we have a CTF Tryhackme - Ignite. Let’s go!!!

Firstly, always, I will scan open port. I use Rustscan because it faster than nmap, moreover it can integrates with nmap.

It just have a open port 80 which running Apache httpd version 2.4.18 on Ubuntu server.

With header Fuel CMS, That’s mean it was built on fuel cms version 1.4. In the robots.txt I have a entry /fuel. This path wiil point to the login page of fuel cms.

And I have a default username and password, so let’s try it.

The dashboard site is not really special. Actually, when I saw the fuel cms and it’s version, I thought that I could found some exploits about it in exploit-db, but this site allowed to login so I tried it first.

With 3 exploits, I chose the python exploit in php.

I got a command line, try to get shell with python or others shell but it doesn’t work at all.

Another way that I will upload php shell from my local machine.

Create a http server with python.

Then use wget on remote machine to download php shell.

Create a listener with netcat and open file on browser http://MACHIEN-IP/shell.php

In www-data user, I got user flag.

Now is privilege escalation. I tried some simple things: sudo -l or crontab or find but nothing happened. So try upload again but now is linPEAS.

Looking around many, many things about the server, I had to check carefully and I got some infomation in fuel_schema database.

Let’s login root with this password.

Oh, I got that. Rooted!
