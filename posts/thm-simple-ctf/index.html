Tryhackme - Simple CTF

September 12, 2021 2-minute read
CTF
ftp • web • linux • php • ssh

Hello, this is Neo. Today we have a CTF of Tryhackme - Simple CTF. Let’s get root!!!

Firstly, always, I will scan open port. I use rustscan because it faster than nmap, moreover it can integrates with nmap.

There’re 3 openning port:

  • Port 21 with ftp-vsftpd 3.0.3
  • Port 80 with web service Apache 2.4.18 on Ubuntu server
  • Port 2222 with ssh service

In port 21, we can login with Anonymous.

However when I try list file, server timeout. I try to search on exploit-db to see if there is any exploit for this version 3.0.3, there is only 1 exploit about DoS.

So let’s check around the site.

I have a robots.txt and 2 disallow enties. But I can’t access it.

However, I found another name: Mike.

Look around, I think the second entry in robots.txt is strange, google and found it. Openemr is a health management software and version 5.0.1.3 has CVE-2018-15139. But it doesn’t seem to work very well because there is no username and password.

Oke. Back to dirsearch.

I found a new path that leads to another cms - CMS Made Simple with version 2.2.8.

I had a CVE-2019-9053, it used SQLi to login without password. Search in exploit-db, I had exploit 46635.

I wonder why in 2019 that guy is still using python2??? Anyway, let’s try to exploit.

And I found the password: secret. Login and take a look around to see if there is any place to upload shells, but this site has a lot of stuff, I’m a bit lazy.

I forgot that I still had port 2222 ssh

And I found the user.txt

In home directory, I have 2 users mitch and sunbath.

Now try to find if there is something running with root.

I have vim, a text editor. So go to GTFOBins and get root.

We have 2 ways to get the root flag. Firstly, using vim to open the file, this way must determine the location of the file, and ordinarily, the root flag will be in the root directory. And the second way is to get root privileges.

With the first way, I use sudo to edit file. Got it.

In the second way, I use sudo in GTFOBins
