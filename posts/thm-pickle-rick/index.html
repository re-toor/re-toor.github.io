Tryhackme - Pickle Rick

September 11, 2021 2-minute read
CTF
web • linux • html

“This Rick and Morty themed challenge requires you to exploit a webserver to find 3 ingredients that will help Rick make his potion to transform himself back into a human from a pickle.”

Hello, this is Neo. Today we have a CTF of Tryhackme - Pickle Rick. Let’s get root!!!

Firstly, always, I will scan open port. I use rustscan because it faster than nmap, moreover it can integrates with nmap.

We have 2 open port:

  • port 22 with ssh service on Ubuntu server
  • port 80 with Apache httpd 2.4.18 on Ubuntu server

Let’s check around the site.

Hmmm… Not special. Let’s check source code.

In here we have a image url and a username.

Before gobuster or feroxbuster or something like that, I will check some popular path: robots.txt, admin, login.php, etc. Passive recon always better.

And I found 2 interesting path robots.txt and login.php.

With robots.txt, I got “Wubbalubbadubdub” I don’t know what it is, so noted that.

In login site, I tried some SQLi or XSS or something but it’s not work.

So for now, we had a login site, a username and a text from robots.txt. So we can try to use the text to password.

Bingo!!!

A simple dashboard. We only use commands, the others need to higher privilege.

Back to Command Panel, I relized that it’s the same like a terminal when I typed “whoami” and it gave me “www-data”.

Listed all file in web dir, I had a strange txt file.

Open it and we had the first flag.

Use “ls” I found 2 user in home directory: rick and ubuntu, in rick I found a file contain the second flag.

But I couldn’t “cat” file, nothing happened. Try a different way with “less” and I found the second flag.

Now, with the last flag, it’s usually in the root directory, that’s mean we have to get the root user.

And as usual, I check “sudo -l”

Tada!! No need password for root

We done. Rooted!!!
